# Project Settings — `tes`

## Overview

The `tes` directory is the **Django project package** (created by `django-admin startproject tes`). It contains the root URL configuration, WSGI/ASGI entry points, and — most importantly — `settings.py`, which controls every aspect of the project.

---

## Files

```
tes/
├── __init__.py
├── asgi.py     — ASGI entry point (for async servers like Daphne/Uvicorn)
├── settings.py — All project configuration
├── urls.py     — Root URL dispatcher
└── wsgi.py     — WSGI entry point (for gunicorn, uWSGI, Vercel)
```

---

## `settings.py` — Detailed Reference

### Environment Variables

Settings are loaded from a `.env` file using `python-dotenv`. Never hard-code secrets.

```python
from dotenv import load_dotenv
load_dotenv()
```

| Environment Variable | Setting | Default |
|----------------------|---------|---------|
| `DJANGO_SECRET_KEY` | `SECRET_KEY` | `"django-insecure-change-me-in-production"` |
| `DEBUG` | `DEBUG` | `False` (must set `"True"` explicitly) |
| `ALLOWED_HOSTS` | `ALLOWED_HOSTS` | `"*"` (splits on comma) |
| `DATABASE_URL` | `DATABASES["default"]` | SQLite fallback |
| `UPI_ID` | `UPI_ID` | `"gpzstore@oksbi"` |
| `PAYEE_NAME` | `PAYEE_NAME` | `"GPZ Store"` |
| `GOOGLE_SCRIPT_URL` | `GOOGLE_SCRIPT_URL` | `""` (sync disabled) |
| `CLOUDINARY_URL` | (read by cloudinary lib) | None — must be set in prod |

### Middleware Stack

```python
MIDDLEWARE = [
    "django.middleware.security.SecurityMiddleware",
    "whitenoise.middleware.WhiteNoiseMiddleware",      # Serve static files efficiently
    "django.contrib.sessions.middleware.SessionMiddleware",
    "django.middleware.common.CommonMiddleware",
    "django.middleware.csrf.CsrfViewMiddleware",
    "django.contrib.auth.middleware.AuthenticationMiddleware",
    "django.contrib.messages.middleware.MessageMiddleware",
    "django.middleware.clickjacking.XFrameOptionsMiddleware",
]
```

**Order matters** — middleware runs top-to-bottom on the request and bottom-to-top on the response:
- `SecurityMiddleware` must be near the top to apply HTTPS redirects/HSTS before any other logic.
- `WhiteNoiseMiddleware` must be before `SessionMiddleware` so static file requests are short-circuited before session processing.
- `CsrfViewMiddleware` must be before any middleware that reads POST data.

### Templates

```python
TEMPLATES = [{
    "BACKEND": "django.template.backends.django.DjangoTemplates",
    "DIRS": [BASE_DIR / "templates"],    # Root-level templates/ folder
    "APP_DIRS": True,                    # Also look in <app>/templates/
    "OPTIONS": {
        "context_processors": [
            "django.template.context_processors.request",
            "django.contrib.auth.context_processors.auth",
            "django.contrib.messages.context_processors.messages",
            "cart.cart.cart_context",    # ← Custom: injects `cart` into every template
        ],
    },
}]
```

**Template resolution order**: `DIRS` paths are checked first, then each app's `templates/` subdirectory (because `APP_DIRS=True`).

### Database

**Production** (when `DATABASE_URL` is set):
```python
DATABASES = {
    "default": dj_database_url.config(
        default=DATABASE_URL,
        conn_max_age=600,        # Persistent connections (10 min)
        conn_health_checks=True,
    )
}
```

**Development** (SQLite fallback):
```python
DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.sqlite3",
        "NAME": BASE_DIR / "db.sqlite3",
    }
}
```

The `dj_database_url` library parses `DATABASE_URL` (e.g. `postgres://user:pass@host/db`) into the correct Django `DATABASES` dict automatically.

### Sessions

```python
SESSION_ENGINE       = "django.contrib.sessions.backends.signed_cookies"
SESSION_COOKIE_HTTPONLY = True
SESSION_COOKIE_SECURE   = not DEBUG   # HTTPS-only in production
SESSION_COOKIE_SAMESITE = "Lax"
```

**Why `signed_cookies`?** Cart data is stored in the session. Using signed cookies means session data is stored client-side (in the browser cookie), avoiding database reads for every request. The downside is that sessions cannot be invalidated server-side (e.g. force-logout all sessions). For the cart use case, this is acceptable.

> If you later add features requiring server-side session invalidation (e.g. revoke all user sessions), switch to `"django.contrib.sessions.backends.db"` and run `python manage.py migrate` to create the sessions table.

### Static Files

```python
STATIC_URL  = "static/"
STATIC_ROOT = BASE_DIR / "staticfiles"     # collectstatic output directory
STATICFILES_DIRS = [BASE_DIR / "static"]   # Additional static files to include
```

**Storage backend:**
- **Production**: `whitenoise.storage.CompressedManifestStaticFilesStorage` — serves files with content-hashed filenames for cache busting.
- **Development**: `django.contrib.staticfiles.storage.StaticFilesStorage` — simple serving without hashing.

```python
STORAGES = {
    "staticfiles": {
        "BACKEND": (
            "whitenoise.storage.CompressedManifestStaticFilesStorage"
            if not DEBUG
            else "django.contrib.staticfiles.storage.StaticFilesStorage"
        ),
    },
}
```

### Media Files (Cloudinary)

```python
MEDIA_URL  = "/media/"
MEDIA_ROOT = BASE_DIR / "media"

STORAGES = {
    "default": {
        "BACKEND": "cloudinary_storage.storage.MediaCloudinaryStorage",
    },
    ...
}
```

All `ImageField` and `FileField` uploads go to Cloudinary. The local `media/` directory is **not used** in production because Vercel's filesystem is read-only. `CLOUDINARY_URL` must be set in `.env`.

**Local development**: Cloudinary will still be used if `CLOUDINARY_URL` is present in `.env`. To use local storage during development, swap `"default"` to `django.core.files.storage.FileSystemStorage`.

## Root URL Configuration (`tes/urls.py`)

The root URL config dispatches requests to each app:

```python
urlpatterns = [
    path("admin/",      admin.site.urls),
    path("",            include("core.urls")),
    path("accounts/",   include("accounts.urls")),
    path("products/",   include("products.urls")),
    path("cart/",       include("cart.urls")),
    path("orders/",     include("orders.urls")),
    path("reviews/",    include("reviews.urls")),
]
```

## Password Validation

Django's built-in validators are all active:

| Validator | What it checks |
|-----------|---------------|
| `UserAttributeSimilarityValidator` | Password is not too similar to username/email |
| `MinimumLengthValidator` | Minimum 8 characters |
| `CommonPasswordValidator` | Not in the list of common weak passwords |
| `NumericPasswordValidator` | Not entirely numeric |

## Internationalization

```python
LANGUAGE_CODE = "en-us"
TIME_ZONE = "UTC"
USE_I18N = True
USE_TZ = True
```

All timestamps stored in the database are UTC. The `TIME_ZONE` setting only affects the `{% now %}` template tag and `timezone.localtime()` calls if `USE_TZ=True`.

## Deployment Notes (Vercel)

- `vercel.json` configures the deployment.
- `build.sh` likely runs `collectstatic` and `migrate`.
- `wsgi.py` exposes `application` — Vercel's Python runtime uses WSGI.
- Static files are served by WhiteNoise from `STATIC_ROOT`.
- Media files (user uploads) go to Cloudinary because Vercel's filesystem is read-only.
- `DEBUG` must be `"False"` in the Vercel environment variables.
- `DJANGO_SECRET_KEY` and `CLOUDINARY_URL` must be set as Vercel environment secrets.
